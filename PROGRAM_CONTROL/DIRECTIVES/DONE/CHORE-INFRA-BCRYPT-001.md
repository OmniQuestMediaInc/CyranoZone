## Objective

Add the `bcrypt` runtime dependency and `@types/bcrypt` type declarations to the top-level `package.json` so that DFSP-001 Module 3 (`PlatformOtpService`) can satisfy its documented Invariant #13 bcrypt carve-out for `OtpEvent.code_hash` hashing.

Claude Code DROID halted DFSP-001 at STEP 2→3 with a BLOCKED report (`9ea9641` on branch `claude/droid-mode-dfsp-001-jiqVt`) because no bcrypt-family or argon2 package was present on main. Scope Discipline correctly prevented a silent cross-directive `package.json` edit. This chore unblocked that dispatch.

---

## Why bcrypt specifically

DFSP Engineering Spec v1.0 Module 3 and DFSP-001.md specify bcrypt. The Invariant #13 carve-out reasoning: SHA-256 is GPU-brute-forceable against the constrained 32⁷ (~34 billion) OTP space on a DB breach; bcrypt's tunable cost factor and per-hash salt mitigate this. Substituting a different primitive would require a DFSP Engineering Spec revision.

The canonical `bcrypt` npm package (native, node-gyp) matches NestJS convention and the `bcrypt.compare()` API the directive expects.
